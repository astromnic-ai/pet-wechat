# 需求文档

## 简介

修复 GitHub Issues #40-#47 中的 7 个 bug（#43 搁置），涵盖主页导航、设置页交互、消息中心样式、数据中心页面功能、宠物动态定制上传等问题。同时为所有修复编写端到端测试覆盖。

## 需求

### 需求 1 - 主页宠物昵称区域跳转到宠物信息编辑（#40 + #41）

**用户故事：** 作为宠物主人，我想要点击主页昵称区域进入宠物信息编辑页，以便快速修改宠物资料

#### 验收标准

1. 当用户已有宠物时，如果点击主页左上角宠物昵称/品种区域（`top-card`），那么应跳转到 `/pages/pet-info/index?petId={当前宠物id}`
2. 当用户没有宠物时，如果点击主页左上角区域，那么应跳转到 `/pages/pet-info/index`（创建新宠物）
3. 当用户已有宠物时，如果点击中间宠物主图（Swiper 中的宠物图片），那么应跳转到 `/pages/pet-avatar/index?petId={当前宠物id}`（定制动态）
4. 当用户没有宠物时，点击中间空白宠物区域的行为保持不变（已有 `handleAddPet` 跳转到 pet-info）

### 需求 2 - 设置页点击反馈（#46）

**用户故事：** 作为用户，我想要点击设置项后获得明确反馈，以便知道该功能暂未开放

#### 验收标准

1. 当用户点击任何暂未开放的设置项（通知设置、隐私设置、主题设置、语言选择、关于我们、帮助与反馈、隐私政策）时，那么应弹出 toast 提示"即将上线，敬请期待"
2. 当用户点击"退出登录"时，退出登录的逻辑保持不变

### 需求 3 - 消息中心返回按钮统一（#45）

**用户故事：** 作为用户，我想要消息中心的返回按钮与其他页面一致，以便获得统一的视觉体验

#### 验收标准

1. 消息中心页面应使用与其他页面相同的 `PageBack` 组件替换当前的文字 `←` 返回按钮
2. 返回按钮的位置和样式应与设置页、数据页等使用 `PageBack` 的页面完全一致

### 需求 4 - 动态记录页月视图（#44）

**用户故事：** 作为宠物主人，我想要在数据页切换到月视图，以便查看更长时间范围的宠物活动数据

#### 验收标准

1. 当用户点击"月"标签时，那么应正常切换到月视图模式
2. 月视图应展示最近 30 天的日活跃度柱状图（横轴为日期，纵轴为活跃度）
3. 月视图应展示该月的活动类型饼图统计
4. 后端 `/api/stats/:petId` 需要新增月度数据支持（`monthBars` 字段）

### 需求 5 - 数据中心 500 错误修复（#42）

**用户故事：** 作为宠物主人，我想要正常访问数据中心查看宠物活动统计

#### 验收标准

1. 当用户从主页进入数据中心时（通过 QuickNav），页面应正常加载并展示数据
2. 当用户没有宠物时，页面应显示"暂无宠物数据"而不是报错
3. 当后端查询失败时，页面应显示友好的错误提示而不是 500 错误
4. 后端 stats 路由应正确处理边界情况（无行为数据、无宠物等）

### 需求 6 - 定制宠物动态上传失败修复（#47）

**用户故事：** 作为宠物主人，我想要正常上传照片定制宠物动态

#### 验收标准

1. 当用户上传照片并点击"开始定制"时，图片应能成功上传到 MinIO 存储
2. 上传成功后应自动创建 avatar 定制任务并跳转到进度页
3. 当上传失败时，应显示具体的错误原因（如"文件格式不支持"、"文件过大"等），而不是笼统的"文件上传失败"
4. 前端 `uploadFile` 工具函数应正确解析后端的错误响应

### 需求 7 - 端到端测试覆盖

**用户故事：** 作为开发者，我需要端到端测试覆盖所有修复的 issue，防止回归

#### 验收标准

1. 后端 API 集成测试：覆盖 stats 路由的正常和边界情况、upload 路由的错误处理
2. 前端 E2E 测试（通过 JS 代码操控微信开发者工具）：覆盖以下完整用户行为
   - 主页昵称区域点击→跳转宠物信息编辑
   - 主页宠物主图点击→跳转定制动态
   - 设置页各项点击→toast 提示
   - 消息中心→返回按钮可用
   - 数据中心→日/周/月视图正常切换
   - 宠物动态定制→上传照片流程
