# 需求文档

## 简介

将前端分享/邀请按钮接通到后端真实邀请 API 和微信原生分享能力。当前这些按钮只弹 Toast 提示，没有实际功能。后端已有完整的邀请码生成（`POST /api/devices/invite`）和接受邀请（`POST /api/devices/invite/:code/accept`）接口。

## 需求

### 需求 1 - devices 页面「分享授权」按钮触发微信分享

**用户故事：** 作为宠物主人，我想要点击项圈卡片上的「分享授权」按钮后直接弹出微信原生分享面板，以便快速将宠物授权分享给亲友。

#### 验收标准

1. 当用户在 devices 页面点击项圈卡片上的「分享授权」按钮时，系统应该调用 `POST /api/devices/invite` 生成邀请码，并弹出微信原生分享面板
2. 当分享面板弹出时，分享卡片标题应包含宠物名称（如「xxx邀请你一起看xxx」），分享路径应为 `/pages/invite/index?code={inviteCode}`
3. 当 API 调用失败时，系统应该显示错误提示，不弹出分享面板

### 需求 2 - devices 页面桌面端「分享」按钮触发微信分享

**用户故事：** 作为宠物主人，我想要点击桌面端卡片上的「分享」按钮后弹出微信分享面板，以便将宠物桌面端权限分享给亲友。

#### 验收标准

1. 当用户在 devices 页面点击桌面端卡片上 action 为 "share" 的按钮时，系统应该调用 `POST /api/devices/invite` 生成邀请码，并弹出微信原生分享面板
2. 分享卡片内容与需求 1 一致（同一宠物的邀请码）
3. 非 "share" action 的按钮（删除权限、解绑）保持现有行为不变

### 需求 3 - invite 页面「一键连接」按钮触发微信分享

**用户故事：** 作为宠物主人，我想要在 invite 页面点击「一键连接」按钮后弹出微信分享面板，以便通过分享链接邀请亲友。

#### 验收标准

1. 当用户在 invite 页面（无 code 参数模式）点击「一键连接」按钮时，系统应该调用 `POST /api/devices/invite` 生成邀请码，并弹出微信原生分享面板
2. 当宠物数据尚未加载完成时，分享按钮不应触发分享（避免无效邀请码）
3. 分享卡片标题和路径与需求 1 一致

### 需求 4 - 分享落地页（invite 页面 code 模式）

**用户故事：** 作为被邀请的亲友，我点击分享卡片进入小程序后，想要看到邀请信息并一键接受授权。

#### 验收标准

1. 当接收方通过分享卡片打开小程序时，系统应该跳转到 `/pages/invite/index?code={inviteCode}`，显示现有的接受邀请 UI
2. 现有的接受邀请流程（调用 `POST /api/devices/invite/:code/accept`）保持不变，无需修改

## 技术约束

- 微信小程序中，分享必须通过 `<Button openType="share">` 触发 + `useShareAppMessage` 定义分享内容
- `useShareAppMessage` 回调支持返回 Promise（微信基础库 >= 2.12.0），可在回调中异步调用 API
- 需要将现有 `<View>` 分享按钮替换为 `<Button openType="share">`，并通过 CSS 重置按钮默认样式
