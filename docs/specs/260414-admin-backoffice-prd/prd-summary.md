# 0.8 宠物在场 - 桌面摆台 PRD（管理后台范围摘录）

来源：`/Users/yangjunior/.hermes/cache/documents/doc_ff3880530b95_0.8宠物在场-桌面摆台 PRD (1).pdf`

本文仅提取“后端管理功能”相关需求，供 `/dev` 开发流程使用。

## 目标
为 YEHEY 宠物“在场”项目补全管理后台，使运营/管理员可以：
- 查看系统概览、设备活跃和实时动态
- 配置按宠物类型划分的系统行为日程
- 审核用户上传的宠物图片
- 管理定制任务和动作素材上传进度
- 查看数据看板
- 检索和管理设备
- 查看用户与会员信息

## 模块范围与优先级

### P0
1. 系统概览 / 数据汇总
2. 行为日程管理
3. 图像审核中心
4. 定制中心
5. 设备管理（搜索筛选、列表）

### P1
1. 系统概览中的设备分布、设备活跃、实时动态
2. 数据看板
3. 设备详情管理
4. 用户管理基础信息

### P2
1. 系统健康状态
2. 用户会员权益配置

## 详细需求

### 1. 系统概览
- 汇总总用户：至少覆盖注册用户、已绑定设备用户、已定制用户、使用项圈用户
- 近一周活跃设备数
- 今日实时互动总数（PRD 中写“实时更新/每天更新”，实现时可先采用近实时或轮询）
- 设备分布：桌面端 vs 项圈数量与占比
- 设备活跃：基于 3 天周期划分高/中/低活跃度
- 实时动态：
  - 新设备上线（每小时更新）
  - 长时间离线设备（每天更新）
  - 新增图像审核（每小时）
  - 个性化定制（每天）
  - 电量预警（当前主要针对项圈）
- 系统健康状态：服务器、API、数据库、云存储空间（可先占位）

### 2. 宠物行为日程管理
- 按宠物类型配置每日活动时间轴：猫、狗、其他
- 动作管理：
  - 6 种基础动作：蹲坐、吃饭、睡觉、趴卧、跑、走
  - 8 种趣味动作：玩球、噗噗、看电视、追尾巴、挠空气、做美梦、舔爪子/wink、转圈
- 支持时间块编辑（拉伸/缩小）、动作拖拽、时间段滑动查看
- 生效策略：每天、工作日、节假日等
- 保存配置后可选择应用当前配置

### 3. 图像审核中心
- 顶部任务统计：
  - 今日待处理
  - 今日已完成
  - 已同步设备总数
- 审核池按分类展示：待审核 / 已通过 / 已拒绝
- 审核项信息至少包含：图片缩略图、宠物信息、上传时间、审核状态
- 审核操作：
  - 通过：进入定制池 + 推送通知
  - 拒绝：可选择通用原因或手动填写原因
  - 下载：支持下载到本地
- 审核结果需同步更新到客户端侧状态/消息

### 4. 定制中心
- 显示累计定制总量与“今日新增”待定制数量
- 定制任务列表采用卡片式展示，至少包含：
  - 上传原图
  - 用户信息
  - 用户 ID
  - 宠物信息
  - 定制分类
- 支持筛选定制类型（基础图像 / 个性化定制）
- 选中任务后，右侧工作区展示宠物定制进度与内容分类
- 动作定制池：
  - 展示该宠物所有动作的定制进度
  - “+” 上传素材
  - “✔” 表示完成
  - 支持动作占位图预览
- 支持填写宠物描述文案
- 完成所有动作定制后，可一键同步到用户端
- 任务状态至少区分：待处理 / 进行中 / 已完成

### 5. 数据看板
- 实时在线设备数（按 1 分钟内在线心跳定义）
- 平均互动数值 = 互动总数 / 在线用户总量
- 平均活跃数值 = 项圈设备行为活跃度 / 当天在线且使用项圈的用户总量
- 一周内累计互动数值排行榜（用户或设备）
- 互动模式分布（当天在线用户的宠物显示模式）

### 6. 设备管理
- 支持组合搜索/筛选：
  - 设备型号（桌面宠物 1.0、宠物项圈 1.0）
  - 宠物图像状态（已上传 / 未上传）
  - 设备绑定状态（已绑定 / 未绑定）
  - 宠物类型（猫 / 狗 / 其他）
  - 设备状态（在线 / 离线）
  - 注册时间（顺序 / 倒序）
- 设备列表显示所有设备详情，支持实时刷新
- 设备详情页：
  - 查看设备底层信息
  - 预留编辑与永久删除功能

### 7. 用户 / 会员管理
- 展示微信同步信息（头像、昵称）
- 展示账号状态、注册时间
- 会员内容先作为占位，但后续要支持：
  - 会员等级
  - 到期时间
  - 绑定设备上限
  - 定制次数配置修改

## 建议的首期交付策略
为降低一次性交付复杂度，建议首个 PR 采用“可用骨架 + Mock/真实接口混合”的方式：

### 首期必须落地（建议）
- 管理后台信息架构与导航
- 系统概览页面（优先核心 KPI + 基础卡片）
- 行为日程页面（至少支持查看、编辑、保存一套时间轴配置）
- 图像审核页面（列表 / 详情 / 通过 / 拒绝）
- 定制中心页面（任务列表、动作进度、上传入口、状态流转）
- 设备管理页面（筛选、列表、详情）
- 用户管理页面（基础信息）

### 可先占位或延后
- 系统健康状态
- 完整会员权益配置
- 极复杂实时能力（可先用轮询 / mock）
- 真正的大文件上传与对象存储联通（可先预留接口）

## 产品决策偏好（供实现时参考）
- 优先做 P0 闭环，P1 做能展示价值的最小版本，P2 可占位
- 若后端接口不足，可优先补 admin 所需 API / mock fallback，而不是把前端卡死
- 若 PRD 文案存在歧义，优先选择“能支撑运营后台日常操作”的保守设计
- 对“实时”需求默认采用可接受的轮询方案，而不是强依赖 websocket
